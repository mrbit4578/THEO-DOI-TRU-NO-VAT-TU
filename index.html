🔑 Hướng dẫn lấy GitHub Token cho 5 user

Mỗi user cần Personal Access Token (PAT) để đọc/ghi file data/store.json trên repo mrbit4578/THEO-DOI-TRU-NO-VAT-TU.

Cách A — Fine-grained token (khuyến nghị, an toàn hơn)

  1. Đăng nhập GitHub → bấm avatar góc phải → Settings.
  2. Cuối menu trái: Developer settings.
  3. Personal access tokens → Fine-grained tokens → Generate new token.
  4. Đặt tên: truno-user1 (hoặc user2…5), hạn dùng 30–90 ngày.
  5. Resource owner: chọn tài khoản mrbit4578 (hoặc org sở hữu repo).
  6. Repository access: Only select repositories → chọn THEO-DOI-TRU-NO-VAT-TU.
  7. Permissions → Repository permissions:
    • Contents: Read and write
    • Metadata: Read-only (tự bật)
  8. Bấm Generate token → copy ngay (chỉ hiện 1 lần).

Cách B — Classic token (nhanh)

  1. Developer settings → Personal access tokens → Tokens (classic).
  2. Generate new token (classic).
  3. Tick scope: repo (full control of private repositories). Nếu repo public và chỉ cần contents, có thể dùng fine-grained như Cách A.
  4. Generate → copy token.

Đưa token cho từng user

  1. Gửi token riêng qua kênh nội bộ (không chat public, không commit vào git).
  2. User mở web app → bấm 🔑 Token → dán token → Lưu token.
  3. Token chỉ lưu trên trình duyệt máy đó (localStorage), không upload đi đâu khác ngoài GitHub API.
  4. Chọn User 1…5 → Tải GitHub → sửa → Lưu GitHub.
Mẹo 5 user: tạo 5 fine-grained token (user1…user5), cùng quyền Contents R/W trên 1 repo. Khi user nghỉ việc → revoke token đó trong GitHub Settings.
Cảnh báo: Ai có token + quyền write đều sửa được store.json. Không đưa token cho người ngoài. Không chụp màn hình token lên group chat.
